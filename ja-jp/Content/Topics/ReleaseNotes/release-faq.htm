リリースに関するよくある質問

目次

  1. 最新のリリースはどのバージョンで、次回のリリースはいつですか。
  2. orgに適した現在のバージョンとorgタイプはどこで見つかりますか。
  3. Oktaは製品機能、拡張機能、修正をどのような方法でリリースしますか。
  4. 月次リリースと週次アップデートの命名規則はどのようになっていますか。
  5. プレビューと本番環境のリリースの違いは何ですか。
  6. 早期アクセス機能と一般利用可能機能とは何ですか。
  7. 特定のリリースのリリースノートを見つけるには、どうすればよいですか。

2. orgに適した現在のバージョンとorgタイプはどこで見つかりますか。

    Okta管理コンソールにサインインします。ページフッターに自分のorgの情報が次のように表示されます。
    • リリースバージョン
    • orgタイプ：プレビューまたは本番環境
    • セル番号

3. Oktaは製品機能、拡張機能、修正をどのような方法でリリースしますか。

    月次リリースは月に一度提供され、次のような内容になっています。

    • 一般利用可能（GA）および早期アクセス（EA）機能
    • 機能強化
    • 修正
    • 新しいアプリ統合

    週次アップデートは、月次リリース後にデプロイされ、次のような内容になっています。

    • 影響の小さい一般的な修正とアプリ統合

4. 月次リリースと週次アップデートの命名規則はどのようになっていますか。

    月次リリースと週次アップデートでは、デプロイメントの日付に基づいて以下の規則に従っています。

    <年>.<月>.<リリースタイプ>

    例：

    • 2022.04.0 - 2022年4月の月次リリース（.04 = 月番号、.0 = 月次リリース）
    • 2022.04.2 - 2022年4月の週次アップデート（.04 = 月番号、.2 = 月次リリース後2回目の週次アップデート）

5. プレビューと本番環境のリリースの違いは何ですか。

    • プレビューリリース：機能、強化、修正、統合がプレビューorgにのみデプロイされます。
    • 本番環境リリース：機能、強化、修正、統合が本番orgにデプロイされます。

    注意：プレビューリリースで導入されるほとんどの機能と強化は通常、次回の月次リリースの本番環境ビルドで使用可能になります。

6. 早期アクセス機能と一般利用可能機能とは何ですか。

    早期アクセス（EA）：Oktaサポートへのリクエストで使用可能になるオプトイン機能。

    スーパー管理者は、管理コンソールの［Settings（設定）］>［Features（機能）］で一部のEA機能を有効または無効にできます。

    一般利用可能（GA）： お客様のSKUに基づいて自動的にすべてのorgで使用可能になる機能。
