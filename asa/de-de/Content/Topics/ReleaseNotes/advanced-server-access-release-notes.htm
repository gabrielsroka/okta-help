Advanced Server Access-Client

Version: 1.61.0

Bereitstellungsdatum: 15. Juni 2022

Die folgenden Betriebssysteme werden von Advanced Server Access nicht mehr unterstützt:

  • Ubuntu 12.04
  • Red Hat Enterprise Linux 6

Eine Liste der unterstützten Optionen finden Sie unter Unterstützte Betriebssysteme.

Verbesserungen

  • Der Advanced Server Access-Client kann Bash- und Zsh-Befehle automatisch vervollständigen.

Frühere Versionen

Version: 1.60.0

Bereitstellungsdatum: 4. Mai 2022

Ab Juni 2022 werden die folgenden Betriebssysteme von Advanced Server Access nicht mehr unterstützt:

  • Ubuntu 12.04
  • Red Hat Enterprise Linux 6

Wenn Sie eines der aufgeführten Betriebssysteme verwenden, empfiehlt Okta, bis Juni 2022 ein Upgrade auf ein unterstütztes Betriebssystem durchzuführen. Siehe Unterstützte Betriebssysteme.

Verbesserungen

  • Der Befehl sft ssh kann die Umgebungsvariable SFT_ALLOW_INSECURE_USERNAMES=1 enthalten, um SSH-Verbindungen mit Benutzernamen, die Sonderzeichen enthalten, temporär zuzulassen. Siehe Verwendung des Advanced Server Access-Clients.
  • Der Client unterstützt jetzt den Algorithmus Diffie Hellman Group Exchange SHA256 Kex Algorithm.
  • An der AD-Einbindung wurden Änderungen vorgenommen. Siehe AD-Einbindung.
    • Wenn Benutzer mit mehreren AD-Konten eine Verbindung zu einem erkannten AD-Gerät herstellen, können sie auswählen, welches Konto sie dabei verwenden.
    • Benutzer können Verbindungen zu AD-Geräten herstellen, ohne ein Kennwort einzugeben. Siehe Kennwortlose Zertifikate.
    • Benutzer können sowohl unter macOS als auch unter Windows über die grafische Benutzeroberfläche Verbindungen zu AD-Geräten herstellen.
  • Benutzer können die Option client.timeout_seconds konfigurieren, um die Zeit festzulegen, wie lange höchstens auf eine Antwort von Servern gewartet wird. Siehe Client-Konfiguration.
  • Nicht registrierte Client geben jetzt detailliertere Fehlermeldungen zurück.

Fehlerbehebungen

  • Die Festlegung eines Benutzernamens mit SSH-Verbindungen gab einen Fehler zurück.
  • Der Link zur Advanced Server Access-Dokumentation war beim macOS-Client falsch.
  • Wenn das Team die Advanced Server Access-API-Rate oder das Parallelitätslimit bereits überschritten hatte, versuchten Clients bei bestimmten Vorgängen nicht, die Verbindung erneut herzustellen.
  • Die Umgebungsvariable SFT_Hook_Events für Lebenszyklus-Hooks war für Skripte nicht verfügbar.

Version: 1.51.3

Bereitstellungsdatum: 23. März 2021

Verbesserungen

  • Der Server-Agent validiert jetzt Änderungen an der Datei sshd_config. Dadurch wird verhindert, dass eine falsch formatierte Konfigurationsdatei die Konnektivität beeinträchtigt.
  • Der Server-Agent erstellt jetzt eine Sicherungskopie der Datei sshd_config, bevor er sie ändert. So wird sichergestellt, dass eine funktionierende Konfiguration im Falle einer ungültigen Konfigurationsdatei wiederhergestellt werden kann.
  • Die standardmäßige Thread-Anzahl für Computer mit vielen Prozessoren wurde gesenkt. Die Leistung der einzelnen Prozessoren wird damit verbessert.

Fehlerbehebungen

  • Wenn On-Demand-Benutzer versuchten, ein RDP-Verbindung zu einem Windows-Server herzustellen, schlug der erste Versuch manchmal fehl.
