Erstellen eines Zertifikats mit einer Zertifikatsignaturanforderung

Eine Zertifikatsignaturanforderung enthält Informationen, die zum Erstellen einer untergeordneten Zertifizierungsstelle in einer Active Directory (AD)-Domäne benötigt werden.

Teams müssen die Zertifikatsignaturanforderung in Advanced Server Access erstellen und mit einer AD-Domäne signieren. Anschließend können die Teams das signierte Zertifikat zu Advanced Server Access hochladen und es mit beliebig vielen AD-Verbindungen verknüpfen. Je nach AD-Umgebung können die folgenden Schritte z. T. abweichend sein:

Bevor Sie beginnen

  • Installieren Sie die AD-Zertifikatdienste als Stammzertifizierungsstelle für die Domäne. Siehe Installation der Zertifizierungsstelle.

Mit dieser Aufgabe beginnen

  1. Erstellen Sie eine Zertifikatsignaturanforderung in Advanced Server Access.
    1. Öffnen Sie das Dashboard von Advanced Server Access.
    2. Klicken Sie im Benutzermenü auf Team Settings (Team-Einstellungen).
    3. Gehen Sie zur Registerkarte Passwordless Certificates (Kennwortlose Zertifikate). 
    4. Klicken Sie auf Erstellen > Certificate Signing Request (Zertifikatsignaturanforderung).
    5. Im Fenster „Create Certificate Signing Request“ (Zertifikatsignaturanforderung erstellen) konfigurieren Sie dann die Einstellungen für das Zertifikat.
    6. Klicken Sie auf Create Certificate Signing Request (Zertifikatsignaturanforderung erstellen).
      Ihr Gerät lädt die Zertifikatsignaturanforderung herunter. Verschieben Sie diese Datei zu einem AD-Domänencontroller.
  2. Erstellen Sie eine Zertifikatvorlage auf einem AD-Zertifikatdienste-Server. 
    1. Öffnen Sie die Zertifizierungsstelle.
    2. Klicken Sie mit der rechten Maustaste auf Zertifikatvorlagen, und wählen Sie Verwalten aus. Das Fenster Zertifikatvorlagenkonsole wird geöffnet.
    3. Klicken Sie mit der rechten Maustaste auf Untergeordnete Zertifizierungsstelle, und wählen Sie Vorlage duplizieren aus. Das Fenster „Eigenschaften“ wird geöffnet.
    4. Geben Sie auf der Registerkarte „Allgemein“ unter „Anzeigename der Vorlage“ ADPasswordless ein.
    5. Wählen Sie auf der Registerkarte „Erweiterungen“ Anwendungsrichtlinie aus, und klicken Sie anschließend auf Bearbeiten. Das Fenster Anwendungsrichtlinienerweiterung bearbeiten wird geöffnet.
    6. Klicken Sie auf Hinzufügen. Das Fenster Anwendungsrichtlinie hinzufügen wird geöffnet.
    7. Wählen Sie Smartcard-Anmeldung und Clientauthentifizierung aus, und klicken Sie dann auf OK.
    8. Klicken Sie im Fenster Anwendungsrichtlinienerweiterung bearbeiten auf OK.
  3. Stellen Sie ein Zertifikat aus. 
     Hinweis: Passen Sie die folgenden Befehle an den Namen Ihrer Zertifikatsignaturanforderung an.
    1. Kehren Sie zur Zertifizierungsstelle zurück.
    2. Klicken Sie mit der rechten Maustaste auf Zertifikatvorlagen, und gehen Sie zu Neu > Auszustellende Zertifikatvorlage .
    3. Wählen Sie im Fenster Zertifikatvorlagen aktivieren die erstellte Zertifikatvorlage aus, und klicken Sie auf OK.
    4. Öffnen Sie eine Eingabeaufforderung als Administrator.
    5. Navigieren Sie zu dem Verzeichnis, in dem Sie die Zertifikatsignaturanforderung gespeichert haben, und geben Sie den folgenden Befehl ein:
      certreq -attrib "CertificateTemplate:ADPasswordless" -submit "ZERTIFIKATSIGNATURANFORDERUNG_IHRES_TEAMS" "IHR_GESPEICHERTES_ZERTIFIKAT"
    6. Wählen Sie im Bestätigungsfenster die zugehörige Zertifizierungsstelle aus, und klicken Sie auf OK.
      Der AD-Zertifikatdienst stellt ein Zertifikat aus und speichert eine Zertifikatsdatei mit Base-64-Verschlüsselung.
  4. Importieren Sie das Zertifikat in den Speicher Vertrauenswürdige Stammzertifizierungsstellen.
    1. Doppelklicken Sie auf das ausgestellte Zertifikat.
    2. Klicken Sie im Zertifikatfenster auf Zertifikat installieren.
    3. Klicken Sie im Fenster Zertifikatimport-Assistent auf Weiter.
    4. Wählen Sie Alle Zertifikate in folgendem Speicher speichern aus, und klicken Sie auf Durchsuchen.
    5. Wählen Sie im Fenster „Durchsuchen“ Vertrauenswürdige Stammzertifizierungsstellen aus, und klicken Sie auf OK.
    6. Klicken Sie auf Weiter.
    7. Überprüfen Sie die Angaben, und klicken Sie auf Fertig stellen.
      AD importiert das Zertifikat.
  5. Importieren Sie das Zertifikat in AD.
    Hinweis: Passen Sie die folgenden Befehle an den Namen der gespeicherten Zertifikatsdatei an.
    1. Kehren Sie zur Eingabeaufforderung zurück, und navigieren Sie zu dem Verzeichnis, in dem Sie das Zertifikat gespeichert haben.
    2. Veröffentlichen Sie das Zertifikat mit dem folgenden Befehl im DS-Unternehmensspeicher:
      certutil -dspublish -f IHR_GESPEICHERTES_ZERTIFIKAT NTAuthCA
  6. Laden Sie das signierte Zertifikat zu Advanced Server Access hoch.
    1. Kehren Sie zur Registerkarte Passwordless Certificates (Kennwortlose Zertifikate) auf dem Advanced Server Access-Dashboard zurück.
    2. Identifizieren Sie den erstellten Zertifikatdatensatz.
    3. Klicken Sie auf > Zertifikat hochladen.
    4. Klicken Sie im Fenster „Zertifikat hochladen“ auf Dateien durchsuchen.
    5. Suchen Sie im Datei-Explorer-Fenster die ausgestellte Zertifikatsdatei.
    6. Klicken Sie im Fenster „Zertifikat hochladen“ auf Hochladen.

Wenn Sie das Zertifikat hochgeladen haben, überprüft Advanced Server Access die Gültigkeit der Zertifikatsdatei. Wenn das Zertifikat gültig ist, können Teams das Zertifikat zu neuen oder vorhandenen AD-Verbindungen hinzufügen. Teams müssen das Zertifikat noch verteilen und mit einer Gruppenrichtlinie die Einstellungen konfigurieren.
