Konfiguration von Gruppenrichtlinien für AD-Server

Nachdem Teams ein Zertifikat in Active Directory (AD) importiert haben, müssen sie noch Serverressourcen für die Verwendung mit Advanced Server Access konfigurieren. Okta empfiehlt, alle durch Advanced Server Access geschützten Server in einer eigenen Organisationseinheit getrennt von anderen Ressourcen in der Domäne zu platzieren. Team können dann Domänencontroller und -ressourcen in der Organisationseinheit mit einem Gruppenrichtlinienobjekt zentral verwalten.

Achtung: Teams sollten die Smartcard-Authentifizierung erst auf Domänencontroller anwenden, wenn die kennwortlose Authentifizierung gründlich getestet wurde. Andernfalls könnte es passieren, dass Teams aus der Domäne ausgeschlossen werden.

Bevor Sie beginnen

  • Erstellen eines selbstsignierten Zertifikats
  • Erstellen eines Zertifikats mit einer Zertifikatsignaturanforderung

Mit der Aufgabe beginnen

  1. Verteilen Sie das Zertifikat mit einer Gruppenrichtlinie.
    1. Klicken Sie mit der rechten Maustaste auf das Domänencontroller-Objekt, und wählen Sie Gruppenrichtlinienobjekt hier erstellen und verknüpfen... aus.
    2. Legen Sie im Fenster „Neues Gruppenrichtlinienobjekt“ als Name für die Gruppenrichtlinie Advanced Server Access – Zertifikat fest.
    3. Klicken Sie auf OK.
    4. Gehen Sie in der Konsolenstruktur zu Computerkonfiguration\Richtlinien\Windows-Einstellungen\Sicherheitseinstellungen\Richtlinien öffentlicher Schlüssel.
    5. Klicken Sie mit der rechten Maustaste auf Vertrauenswürdige Stammzertifizierungsstellen und anschließend auf Importieren.
    6. Klicken Sie unter Willkommen auf Weiter.
    7. Geben Sie den Pfad der Zertifikatsdatei an, die Sie zuvor erstellt haben, und klicken Sie dann auf Weiter.
    8. Klicken Sie auf Alle Zertifikate in folgendem Speicher speichern und dann auf Weiter.
    9. Überprüfen Sie die Angaben, und klicken Sie auf Fertig stellen.
  2. Deaktivieren Sie die Authentifizierung auf Netzwerkebene.
    1. Suchen Sie auf dem Bildschirm „Gruppenrichtlinienverwaltung“ das Gruppenrichtlinienobjekt Advanced Server Access – Zertifikat.
    2. Klicken Sie mit der rechten Maustaste auf das Gruppenrichtlinienobjekt, und wählen Sie Bearbeiten aus.
    3. Gehen Sie in der Konsolenstruktur zu Computerkonfiguration\Richtlinien\Administrative Vorlagen\Windows-Komponenten\Remotedesktopdienste\Remotedesktop-Sitzungshost\Sicherheit.
    4. Doppelklicken Sie auf Benutzerauthentifizierung mit Authentifizierung auf Netzwerkebene ist für Remoteverbindungen erforderlich.
    5. Wählen Sie im Fenster „Eigenschaften“ Deaktiviert aus.
    6. Klicken Sie auf Anwenden und anschließend auf OK.
  3. Weisen Sie das Gruppenrichtlinienobjekt der Advanced Server Access-Organisationseinheit zu.
    1. Suchen Sie im Snap-In „Gruppenrichtlinienverwaltung“ die Advanced Server Access-Organisationseinheit.
    2. Klicken Sie mit der rechten Maustaste auf den Domänencontroller, und wählen Sie Vorhandenes Gruppenrichtlinienobjekt verknüpfen aus.
    3. Wählen Sie im Fenster Gruppenrichtlinienobjekt auswählen das Gruppenrichtlinienobjekt Advanced Server Access – Zertifikat aus.
    4. Klicken Sie auf OK.
  4. Legen Sie fest, dass die Smartcard-Authentifizierung für Konten erforderlich ist.
    • Wenden Sie diese Richtlinie erst auf Domänencontroller an, wenn die kennwortlose Authentifizierung gründlich getestet wurde. Andernfalls könnte es passieren, dass Sie aus der Domäne ausgeschlossen werden.
    • Falls Ihre Okta-Organisation die delegierte Authentifizierung von Active Directory nutzt, können Benutzer nicht mehr auf das Okta-Dashboard zugreifen, wenn Sie diese Richtlinie auf Benutzerkonten anwenden.
    1. Klicken Sie mit der rechten Maustaste auf die Advanced Server Access-Organisationseinheit, und wählen Sie Gruppenrichtlinienobjekt hier erstellen und verknüpfen... aus.
    2. Legen Sie im Fenster „Neues Gruppenrichtlinienobjekt“ als Name für die Gruppenrichtlinie Advanced Server Access – Authentifizierung fest.
    3. Klicken Sie auf OK.
    4. Gehen Sie in der Konsolenstruktur zu Computerkonfiguration\Richtlinien\Windows-Einstellungen\Sicherheitseinstellungen\Lokale Richtlinien\Sicherheitsoptionen.
    5. Doppelklicken Sie auf Interaktive Anmeldung: Windows Hello for Business oder Smartcard erforderlich.
      Hinweis: Einige Windows-Versionen verwenden stattdessen Interaktive Anmeldung: Smartcard erforderlich.
    6. Wählen Sie im Fenster „Eigenschaften“ Diese Richtlinieneinstellung definieren aus.
    7. Wählen Sie Aktiviert aus.
      Achtung: Teams sollten die Smartcard-Authentifizierung erst auf Domänencontroller anwenden, wenn die kennwortlose Authentifizierung gründlich getestet wurde. Andernfalls könnte es passieren, dass Teams aus der Domäne ausgeschlossen werden.
    8. Klicken Sie auf Anwenden und anschließend auf OK.
