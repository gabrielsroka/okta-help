Konfiguration von Agenten-Lebenszyklusverwaltungs-Hooks für Advanced Server Access

Die Funktion für Lebenszyklusverwaltungs-Hooks von Advanced Server Access stellt einen EventBus auf Ihren Servern bereit, der Benachrichtigungen erhält, wenn Mitgliedschaftsänderungen an lokalen Benutzer- und Gruppen-Konfigurationen auf Ihren Linux-Systemen vorgenommen werden. Advanced Server Access-Administratoren können diese Lebenszyklus-Hooks mit Skripten an alle Routineaufgaben anpassen, die ausgeführt werden müssen, wenn es zu solchen Änderungen kommt. Lokale Skripte werden auch global ausgeführt, in Übereinstimmung mit den Dateien im Verzeichnis /usr/lib/sftd/hooks. Sie können erkennen, welche Betriebssystem-Objekte je nach Ereignis betroffen sind und welche Skripte gerade ausgeführt werden. Siehe Verzeichnisse und Skripte und Zum Lebenszyklus gehörige Umgebungsvariablen.

Verfahren

Die folgenden Schritte sollen als Leitfaden für die Einrichtung auf dem Host dienen.

Bevor Sie beginnen, stellen Sie sicher, dass Sie eine aktive Advanced Server Access-Sitzung geöffnet haben und auf dem Linux-Zielserver über Administratorrechte verfügen.

Erstellen eines benutzerdefinierten Skripts

  1. Stellen Sie über die Befehlszeile mit SSH eine Verbindung zu einem der Linux-Zielserver her.

  2. Wechseln Sie mit dem Befehl sudo su zum Root.

  3. Erstellen Sie mit dem folgenden Befehl ein neues Verzeichnis für das Hooks-Skript:

    mkdir -p /usr/lib/sftd/hooks/user-created.d

  4. Erstellen Sie mit dem Texteditor eine neue Datei mit dem Namen 01-copyfiles.sh, und fügen Sie das folgende Bash-Skript in diese Datei ein:

    Hinweis: Das nachfolgende Beispiel zeigt die grundlegende Funktionalität von Agenten-Lebenszyklus-Hooks. Das Skript können Sie an Ihre spezifische Integration anpassen.

    #!/bin/bash

    SOURCE_FILES=/opt/sourcefiles

    cp $SOURCE_FILES/App?/home/"${SFT_HOOK_USERNAME}"

    if [ $? -eq 0 ]

    then

        echo "Quelldateien erfolgreich in /home/"${SFT_HOOK_USERNAME} kopiert"" >> /tmp/sourcefiles.out

    else

        echo "Fehler beim Kopieren der Quelldateien in /home/"${SFT_HOOK_USERNAME}"" >> /tmp/sourcefiles.out

  5. Gewähren Sie den Zugriff für die Skriptausführung:

    chmod 755 01-copyfiles.sh

  6. Führen Sie den Befehl ls -l aus, und überprüfen Sie, ob die Berechtigungen korrekt festgelegt sind. Die Ausgabe sollte in etwa wie folgt aussehen:

    -rwxr-xr-x 1 root root 316 Jun 30 14:54 01-copyfiles.sh

Erstellen der Quelldateien

  1. Erstellen Sie mit den folgenden Befehlen ein neues Verzeichnis für die Quelldateien:

    cd /opt

    mkdir sourcefiles

  2. Erstellen Sie nun mit den folgenden Befehlen die Quelldateien:

    cd sourcefiles

    touch App1 App2 App3

  3. Führen Sie den Befehl ls aus, und überprüfen Sie, ob die Dateien App1, App2 und App3 erstellt wurden.

Erstellen Sie einen neuen Okta-Benutzer, und weisen Sie ihn einer autorisierten Gruppe zu.

  1. Navigieren Sie als Okta-Administrator zum Admin-Dashboard des Okta-Tenants, der die Advanced Server Access-Anwendung verwaltet.

  2. Navigieren Sie zu Anwendungen, und klicken Sie auf Okta Advanced Server Access.

  3. Klicken Sie auf Assignments (Zuweisungen) und dann auf Gruppen. Merken Sie sich den Namen der Gruppe, der die Anwendung zugewiesen wurde.

  4. Wählen Sie in der Okta-Admin-Konsole Directory (Verzeichnis), Personen und dann Add Person (Person hinzufügen) aus.

  5. Erstellen Sie einen neuen Benutzer mit einem Namen, den Sie sich gut merken können.

  6. Navigieren Sie in der Okta-Admin-Konsole zu Directory (Verzeichnis), und klicken Sie dann auf Gruppen. Wählen Sie hier die Gruppe aus, die im 3. Schritt Advanced Server Access zugewiesen wurde.

Hinzufügen der Gruppe zu Ihrem Projekt

  1. Klicken Sie in der Administrator-Konsole von Okta auf Meine Anwendungen, und wählen Sie dann Okta Advanced Server Access aus.

  2. Klicken Sie auf Projekte.

  3. Klicken Sie auf das Projekt, zu dem Sie die Gruppe hinzufügen möchten.

  4. Klicken Sie auf die Registerkarte Gruppen und dann auf Add Group to Project (Gruppe zu Projekt hinzufügen).

  5. Geben Sie im Feld Gruppe den Namen der hinzuzufügenden Gruppe ein, wählen Sie die Gruppe aus, und klicken Sie dann auf Create Group (Gruppe erstellen).

  6. Um zu überprüfen, ob der Benutzer zum Projekt hinzugefügt wurde, klicken Sie auf die Registerkarte Benutzer für dieses Projekt und suchen den Benutzer.

Überprüfen der Benutzerbereitstellung auf dem Linux-Server

  1. Geben Sie in der Linux-Kommandozeile die folgenden Befehle ein:

    cd /home

    ls -l

  2. Vergewissern Sie sich, dass das Stammverzeichnis für den neuen Benutzer erstellt wurde.

  3. Wechseln Sie mit dem Befehl cd <Benutzername> in das Stammverzeichnis des neuen Benutzers.

    Mit dem Befehl ls können Sie überprüfen, ob die Dateien App1, App2 und App3 im Stammverzeichnis des Benutzers vorhanden sind.

So können Sie mit den Lebenszyklusverwaltungs-Hooks des Advanced Server Access -Agenten Host-local-Automatisierungs-, System- und externe Benutzer-Verzeichnisse in Ihre Benutzer und Gruppen in Okta integrieren.

Weiterführende Informationen

Verzeichnisse und Skripte

In den folgenden Abschnitten wird erklärt, welche Verzeichnisse und Skripte je nach aufgetretenem Benutzer- oder Gruppenereignis aufgerufen werden. Der Ereignistyp wird in der Variable SFT_HOOK_EVENT erfasst. Siehe Zum Lebenszyklus gehörige Umgebungsvariablen.

Benutzer

  • user-created

    Wenn ein Benutzer erstellt wird, werden die Skripte in /usr/lib/sftd/hooks/user-created.d/*.sh ausgeführt.

  • user-updated

    Wenn ein Benutzer aktualisiert wird, werden die Skripte in /usr/lib/sftd/hooks/user-updated.d/*.sh ausgeführt.

  • user-deleted

    Wenn ein Benutzer gelöscht wird, werden die Skripte in /usr/lib/sftd/hooks/user-deleted.d/*.sh ausgeführt.

Gruppe

  • group-created

    Wenn eine Gruppe erstellt wird, werden die Skripte in /usr/lib/sftd/hooks/group-created.d/*.sh ausgeführt.

  • group-updated

    Wenn eine Gruppe aktualisiert wird, werden die Skripte in /usr/lib/sftd/hooks/group-updated.d/*.sh ausgeführt.

  • group-deleted

    Wenn die Bereitstellung einer Gruppe aufgehoben wird, werden die Skripte in /usr/lib/sftd/hooks/group-deleted.d/*.sh ausgeführt.

Skripte, die vom globalen Pfad zurückgegeben werden, sind numerisch sortiert.
