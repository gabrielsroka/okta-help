Configuration de la synchronisation en temps réel avec AWS

Cette fonctionnalité est disponible en accès anticipé. Pour l'activer, contactez le Support Okta.

La synchronisation en temps réel permet à l'Advanced Server Access de s'abonner aux événements d'instance de calcul générés par AWS. En temps normal, la synchronisation d'un serveur s'effectue en temps réel, mais cette fonctionnalité permet à l'Advanced Server Access d'ajouter ou de supprimer des serveurs dès la réception d'une notification. La synchronisation en temps réel fonctionne grâce à l'activation d'AWS EventBridge. Ce dernier envoie des événements webhook à un point de terminaison associé à un compte cloud spécifique au sein d'un projet.

La plupart du temps, les événements sont générés lorsqu'une instance de calcul est lancée ou résiliée. Pour les événements de lancement, l'Advanced Server Access ajoute automatiquement les serveurs au projet associé. Vous devez installer l'agent du serveur de l'Advanced Server Access sur les serveurs découverts avant que les utilisateurs puissent se connecter. Pour les événements de résiliation, l'Advanced Server Access supprime automatiquement le serveur du répertoire des projets.

  • Dix minutes peuvent s'écouler avant que les modifications apportées au serveur se synchronisent avec l'Advanced Server Access.
  • Vous devez configurer des règles différentes pour chaque région AWS dans laquelle un serveur se trouve, sauf si vous utilisez des événements interrégionaux pour centraliser vos événements au sein d'une même région.
  1. Identification des informations du projet.
    1. Accédez à la console d'administration de l'Advanced Server Access.
    2. Accédez à la page Projets et sélectionnez un projet.
    3. Accédez à l'onglet Inscription et sélectionnez le compte AWS.
    4. Sous Contrôle du cycle de vie des instances, cliquez sur Create Client ID & Secret (créer un ID client et une clé secrète).
    5. Dans la fenêtre Client Secret Created (clé secrète client créée), stockez l'ID client et la clé secrète.
      Remarque : une fois la fenêtre fermée, vous ne pourrez plus récupérer cette information. Si vous la perdez, vous devrez générer un nouvel ID client et une nouvelle clé secrète.
    6. Cliquez sur OK.
    7. Prenez note de la Webhook URL (URL Webhook). Vous en aurez besoin ultérieurement.
  2. Création d'une connexion.
    1. Accédez à la console Amazon EventBridge.
    2. Accédez à la page API destinations (destinations d'API).
    3. Accédez à l'onglet Connexions et cliquez sur Create connection (créer une connexion).
    4. Depuis la fenêtre Create connection, saisissez un nom et une description.
    5. Pour le type d'autorisation, sélectionnez Identifiants client OAuth.
    6. Configurez les identifiants.
      ChampAction
      Point de terminaison d'autorisation Saisissez https://app.scaleft.com/v1/oauth_token.
      Méthode HTTP Sélectionnez POST.

      Paramètres HTTP OAuth

      1. Sous Parameter (paramètre), sélectionnez Body field (champ du corps).
      2. Sous Key (clé), saisissez grant_type.
      3. Sous Value (valeur), saisissez client_credentials.
      ID client Saisissez l'ID client que vous aviez récupéré via la console de l'Advanced Server Access.
      Clé secrète client Saisissez la clé secrète client que vous aviez récupérée via la console de l'Advanced Server Access.
    7. Cliquez sur Créer.

  3. Création d'une cible.
    Pour plus d'informations, nous vous invitons à consulter la Documentation AWS.

    1. Accédez à la page Destinations d'API et cliquez sur Create API destination (créer une destination d'API).
    2. Dans la fenêtre Create API destination, saisissez un nom et une description.
    3. Pour le point de terminaison de la destination d'API, saisissez la Webhook URL (URL Webhook) que vous aviez récupérée via l'Advanced Server Access.
    4. Pour la méthode HTTP, sélectionnez POST.
    5. Cliquez sur Use an existing connection (utiliser une connexion existante) et sélectionnez la connexion que vous venez de créer.
    6. Cliquez sur Créer.
  4. Création d'une règle.
    Pour plus d'informations, nous vous invitons à consulter la Documentation AWS.
    1. Accédez à la page Règles et cliquez sur Create rule (créer une règle).
    2. Nommez la règle.
    3. Sous Define pattern (définir un modèle), sélectionnez Event pattern (modèle d'événement), puis sélectionnez Custom pattern (modèle personnalisé).
    4. À côté de Event pattern, cliquez sur Modifier.
    5. Insérez le modèle JSON suivant.
      ModèleDescription
      { "source": ["aws.ec2"], "detail-type": ["EC2 Instance State-change Notification"], "detail": { "state": ["pending", "terminated"] } } Vous envoie une notification lorsqu'une instance EC2 passe à l'état pending (en attente) ou terminated (résiliée).
    6. Pour la cible, choisissez la destination d'API cible que vous venez de créer.
    7. Cliquez sur Créer.

Dès que la règle est créée, AWS envoie une notification à l'Advanced Server Access lorsqu'un événement correspondant se produit. L'Advanced Server Access utilise ses notifications pour mettre à jour le répertoire des serveurs dans le projet associé.
