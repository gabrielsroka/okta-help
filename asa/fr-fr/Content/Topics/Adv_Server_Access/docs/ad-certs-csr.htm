Créer un certificat avec une demande de signature de certificat

Une demande de signature de certificat (CSR) inclut les informations nécessaires à la création d'une autorité de certification (CA) secondaire au sein d'un domaine Active Directory (AD).

Les équipes doivent créer la CSR dans Advanced Server Access et effectuer la signature avec un domaine AD. Les équipes peuvent ensuite charger le certificat signé dans Advanced Server Access et associer le certificat à une ou plusieurs connexions AD. Certaines des étapes suivantes peuvent varier en fonction de votre environnement Active Directory.

Avant de commencer

  • Installez Active Directory Certificate Services en tant qu'autorité de certification racine pour le domaine. Consultez la page Installer l'autorité de certification.

Commencer cette tâche

  1. Créez une demande de signature de certificat dans Advanced Server Access.
    1. Ouvrez le tableau de bord Advanced Server Access.
    2. Dans le menu utilisateur, cliquez sur Team Settings (Paramètres de l'équipe).
    3. Rendez-vous dans l'onglet Certificats sans mot de passe. 
    4. Cliquez sur Créer > Certificate Signing Request (Demande de signature de certificat).
    5. Dans la fenêtre Create Certificate Signing Request (Créer une demande de signature de certificat), configurez les paramètres du certificat.
    6. Cliquez sur Create Certificate Signing Request (Créer une demande de signature de certificat).
      Votre appareil télécharge la CSR. Déplacez ce fichier vers un contrôleur de domaine AD.
  2. Créez un modèle de certificat sur un serveur AD Certificate Services (CS). 
    1. Ouvrez Certification Authority (Autorité de certification).
    2. Faites un clic droit sur Certificate Templates (Modèles de certificat) et sélectionnez Manage (Gérer). La fenêtre Certificate Templates Console (Console de modèles de certificat) s'ouvre.
    3. Faites un clic droit sur Subordinate Certification Authority (Autorité de certification secondaire) et sélectionnez Duplicate Template (Dupliquer un modèle). Une fenêtre de propriétés s'ouvre.
    4. Dans l'onglet Général, sous Template Display Name (Nom d'affichage du modèle), saisissez ADPasswordless.
    5. Dans l'onglet Extensions, sélectionnez Application Policy (Stratégie d'application), puis cliquez sur Modifier. La fenêtre Edit Application Policies Extension (Modifier l'extension des stratégies d'application) s'ouvre.
    6. Cliquez sur Ajouter. La fenêtre Add Application Policy (Ajouter une stratégie d'application) s'ouvre.
    7. Sélectionnez Smart Card Logon (Ouverture de session par carte à puce) et Authentification client, puis cliquez sur OK.
    8. Dans la fenêtre Edit Application Policies Extension (Modifier l'extension des stratégies d'application), cliquez sur OK.
  3. Émettez un certificat. 
     Remarque : modifiez les commandes suivantes pour utiliser le nom de votre CSR.
    1. Retournez dans Certification Authority (Autorité de certification).
    2. Faites un clic droit sur Certificate Templates (Modèles de certificat) et rendez-vous dans Nouveau > Certificate Template to Issue (Modèle de certificat à émettre).
    3. Depuis la fenêtre Enable Certificate Templates (Activer les modèles de certificat), sélectionnez le modèle de certificat que vous avez créé et cliquez sur OK.
    4. Ouvrez une invite de commande en tant qu'administrateur.
    5. Rendez-vous dans le répertoire où vous avez enregistré la CSR et saisissez la commande suivante :
      certreq -attrib "CertificateTemplate:ADPasswordless" -submit "YOUR_TEAM_CSR" "YOUR_SAVED_CERT"
    6. Depuis la fenêtre de confirmation, sélectionnez la CA liée et cliquez sur OK.
      Le service AD CS émet un fichier de certificat et l'enregistre en utilisant un encodage en base 64.
  4. Importez le certificat dans la base Trusted Root Certification Authorities (Autorités de certification racines de confiance).
    1. Double-cliquez sur le certificat émis.
    2. Depuis la fenêtre de certificat, cliquez sur Install Certificate (Installer le certificat).
    3. Depuis la fenêtre Certificate Import Wizard (Assistant Importation du certificat), cliquez sur Suivant.
    4. Sélectionnez Place all certificates in the following store (Placer tous les certificats dans la base suivante) et cliquez sur Parcourir.
    5. Depuis la fenêtre de navigation, sélectionnez Trusted Root Certification Authorities (Autorités de certification racines de confiance) et cliquez sur OK.
    6. Cliquez sur Suivant.
    7. Examinez les informations et cliquez sur Terminer.
      AD importe le certificat.
  5. Importez le certificat dans AD.
    Remarque : modifiez les commandes suivantes pour utiliser le nom du fichier de certificat enregistré.
    1. Retournez dans l'invite de commande et rendez-vous dans le répertoire où vous avez enregistré le certificat.
    2. Publiez le certificat dans le magasin DS Enterprise avec la commande suivante :
      certutil -dspublish -f YOUR_SAVED_CERT NTAuthCA
  6. Chargez le certificat signé dans Advanced Server Access.
    1. Retournez dans l'onglet Certificats sans mot de passe dans le tableau de bord Advanced Server Access.
    2. Identifiez l'enregistrement du certificat créé.
    3. Cliquez sur > Charger un certificat.
    4. Dans la fenêtre Charger un certificat, cliquez sur Parcourir les fichiers.
    5. Dans la fenêtre de l'explorateur de fichiers, localisez le fichier de certificat émis.
    6. Dans la fenêtre Charger un certificat, cliquez sur Charger.

Une fois le certificat chargé, Advanced Server Access vérifie la validité du fichier de certificat. Si le certificat est valide, les équipes peuvent l'ajouter à des connexions nouvelles ou existantes. Les équipes doivent tout de même distribuer le certificat et configurer les paramètres en utilisant une stratégie de groupe.

Étapes suivantes
